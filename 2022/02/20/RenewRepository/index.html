Reset a Git Remote Repository

Needs:

Delete the git remote repository, including all files in the remote repository, all commit records, and make it a brand new empty repository.

Scenes:

  1. To rebuild the original project, you need to clear the original history or all code, but do not create a new remote repository.

  2. A submitted file contains sensitive information (password files, etc.).

  3. Need to delete the remote repository for some reason.

1 Create a separate branch

2 Delete local files

If you are going to empty the remote repository, delete all local files (the files are still useful and can be backed up locally); if you delete some files or ignore sensitive files (such as passwords and other information), you can add ignore rules to the .gitignore file.

delete all local files

3 Re-add local files

4 Commit the current branch

5 Delete the original master branch

If there are other branches, delete them as well

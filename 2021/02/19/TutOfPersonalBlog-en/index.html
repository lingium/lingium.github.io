Build a personal blog with Github Pages and Hexo

The first blog will briefly record the construction process. In fact, there are many online tutorials. I just take a little time to record the process. There are not many pits in the construction. System is MacOS.

Why choose Github Pages and Hexo?

I suddenly wanted to build a blog and play around with front-end development. I use Github Pages mainly because it is free and I don't have to spend money to rent a server.

In Jekyll and Hexo, the two static webpage generators, Jekyll relies on Ruby, the speed of static site generation will be slower and slower as the site content increases, while Hexo relies on node, the main focus of speed, there are a lot of available free open source themes and a very active Chinese community.

Init Github Pages

  1. Register a Github account, note down Username and email address
  2. Follow the official tutorial create a new Repositories, note that the repository must be named as<Username.github.io>.

\(\LaTeX\) Support:

We have installed hexo-renderer-kramed and hexo-math. Then modify _config.yml

per_page: false : Turn off the rendering formula of each page, the access speed of the page without the formula will not be affected, but each file needs to be manually add mathjax: true in the markdown file header

Inline formula: \(\Im, \imath, \jmath, \Bbbk, \ell, \mho, \wp, \Re, \circledS\)

Interline formula: \[ dy/dx, \mathrm{d}y/\mathrm{d}x, \frac{dy}{dx}, \frac{\mathrm{d}y}{\mathrm{d}x}, \frac{\partial^2}{\partial x_1\partial x_2}y \]

Multi-line formula: \[ \begin{align} a &= b + c \label{eq3} \\ x &= yz \label{eq4}\\ l &= m - n \label{eq5} \end{align} \]
